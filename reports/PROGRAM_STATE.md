## What this project is

Testing whether a Vision Transformer's **queries** (each patch's "question"
about other patches) should be **dynamic** instead of one fixed learned matrix.
Nine mechanisms built on one shared skeleton; every mechanism adds its
adjustment through a **zero-init learnable gate α** ("volume knob") so training
starts exactly at the vanilla baseline (enforced by a bitwise equivalence test).

**Codename → idea map** (wandb/run-id vocabulary):
a0 = plain baseline · a1/a2/a3 = query adjusted by content / by position / both
· a4 = TPA, query built on the fly (published baseline) · a5 = re-mix existing
heads' queries by content · b1 = menu of query-styles with routing · b2 =
pooled scene context steers queries · b3 = query generated by an inner
attention pass. Suffix `_replace` = mechanism forced, no residual; `_forced` =
α frozen at 0.3 (fair trial); `mps_` prefix = this local screening program.

## Conclusions so far (details + history in analysis.md)

1. **Adoption law**: whether the optimizer takes up a mechanism tracks its
   init-delta magnitude and gradient concentration; the α-uptake loop is
   bilinear and can cold-start-fail. Adoption ≠ merit (see methodology note in
   analysis.md — corrected after user challenge); accuracy pairs are the only
   promotion currency.
2. **Content beats position**: content-keyed adaptivity adopted; everything
   keyed to static position refused; positional terms minor where combined.
   The project's founding fixed-input-size hook is losing on the query side.
3. **The cheapest mechanism leads**: head-remix (+1.17, only cell above noise,
   tied-best robustness, ~free). An extra plain layer with 25× its params does
   worse.
4. **The routed menu (b1) is a completed 5-way negative result**: refused under
   softmax/sigmoid/prior/4×-init/M=4; competition, init-scale, and
   gradient-dilution explanations all experimentally eliminated.
5. **Steering beats replacing**: residual context ≈ neutral-plus;
   GC-ViT-style replacement is the only below-baseline cell.
6. **Depth division of labor**: local/content mechanisms engage layer 0
   hardest; nested-attention peaks mid-network. Best new idea generated:
   depth-heterogeneous hybrid (remix early, nested middle).

## How to resume

```bash
cd /Users/raphael/querent
nohup caffeinate -is uv run python scripts/run_queue_local.py >> reports/runner.log 2>&1 &
```
That's all: the runner re-reads `reports/queue.txt`, skips everything in
`queue_done.txt`, resumes the interrupted RoPE cell from its local checkpoint
(same wandb run), and keeps auto-committing `mps_screening.md` after each cell.
Edit queue.txt freely before/while running. Each cell ≈ 2.5–3.5 h on MPS.
